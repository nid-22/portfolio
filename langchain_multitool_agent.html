General Info

This project demonstrates how to build a multi-tool product catalogue assistant using LangChain’s latest Agent API. The system processes a clothing product catalog stored in CSV format and enables natural-language search and exploration.

The agent dynamically decides which tools to invoke based on user intent and composes final responses using retrieved data and reasoning provided by an LLM.

Dataset

The product catalog is provided as a CSV containing structured fields such as product id, name, description, category, gender, price, and color. A vector index is built over product descriptions for semantic search.

Download raw CSV: OutdoorClothingCatalog_1000.csv (raw)

Process & Architecture

User queries pass to a LangChain Agent (tool-calling LLM) which selects tools to run: vector search for semantic matches, pandas-based filtering for tabular queries, and comparison logic for side-by-side product comparisons.

Tools added

  • Semantic product recommendations using vector search
  • Product comparison with tabular output
  • Gender-aware outfit generation (topwear, bottomwear, shoes)
  • Natural-language filtering using pandas
  • Error handling via agent middleware

Conclusion

This project provides a reference implementation for combining LangChain agents with vector search and tabular tools to deliver a retail assistant. It can be extended with additional tools (pricing engines, inventory APIs, image search) and adapted to real-world product catalogs.
